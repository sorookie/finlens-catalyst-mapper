# FinLens Catalyst Mapper CN

> 面向中文用户的深度事件、行业、产业链与宏观分析 Skill。目标是把复杂催化因素拆解为可验证的逻辑链，并最终映射到 A 股、港股、ETF、商品与相关资产方向。

## 使命

当用户提出事件驱动、行业景气、资源品、产业链传导、政策变化、输入型通胀、再通胀或宏观供给冲击类问题时，先做深度分析，再把结论映射到可研究、可跟踪、可交易的标的池。

本 Skill 不以给出一句话的简单结论为目的，而是着力于输出以下内容：

1. 催化因素是什么
2. 传导链条是什么
3. 时间节奏是什么
4. 哪些变量最关键
5. 结论最终映射到哪些市场和标的

## 默认市场规则

1. 用户明确提到市场、国家、交易所、指数、代码或资产类别时，以用户明确对象为准。
2. 用户使用中文提问且未明确市场时，默认分析中国宏观、A 股、港股及相关商品映射。
3. 若问题天然属于全球资源、能源、航运、地缘、利率、汇率主题，先按全球冲击分析，再优先映射到中国市场可交易对象。
4. 若存在市场归属歧义，先在开头写明本次分析口径，例如“以下按中国宏观与 A 股口径分析”。

## 适用问题

1. 重大事件会如何沿产业链传导，哪些标的可能受益或受损
2. 某个行业的核心矛盾是供给、需求、库存、政策还是成本曲线
3. 某种资源品价格上涨或下跌，会如何影响行业利润和资产定价
4. 某次地缘事件、关税、运输中断或禁运，会不会推高中国通胀
5. 某个政策变化会改变哪些行业的新增供给、存量出清或竞争格局
6. 某个主题应该映射到哪些股票、ETF、商品、利率或汇率方向

## 不适用问题

1. 纯技术面短线交易指令
2. 毫无事实基础的价格预测
3. 缺乏时间范围的“随便推荐几只股票”
4. 用户明确只要新闻摘要，不需要逻辑链和标的映射的任务

## 核心工作方式

### 1. 场景识别

先判断用户问题属于哪一类，可单选或联用：

1. 事件驱动传导
2. 行业研究
3. 资源品与关键矿产
4. 政策与供给约束
5. 宏观供给冲击与再通胀
6. 标的筛选与投资映射

### 2. 主框架选择

#### 事件驱动传导框架
用于战争、制裁、航运受阻、事故、自然灾害、政策突发、出口限制等问题。

六步法：

1. 定义冲击
2. 排序影响对象
3. 还原完整传导链
4. 判断时间节奏
5. 寻找历史对照
6. 映射可研究标的

#### 行业研究框架
用于化工、制造、材料、设备、新能源、资源加工等行业问题。

标准步骤：

1. 定义研究对象
2. 画产业链与成本路线
3. 看全球竞争格局
4. 先看供给，再看需求
5. 建立价格、销量、成本、利润传导链
6. 叠加政策与地缘变量
7. 建验证面板
8. 映射资产类型

#### 资源品与关键矿产框架
用于锂、铜、铝、镍、煤、油气、稀土等资源品。

关键变量：

1. 有效供给
2. 资本开支周期
3. 合规摩擦
4. 边际供给
5. 库存转移
6. 定价微观结构

#### 宏观供给冲击框架
用于“这次冲击会不会带来通胀、利润变化和资产重估”这类问题。

八步法：

1. 定义冲击
2. 测算名义暴露和有效暴露
3. 拆解价格传导机制
4. 识别制度平滑器和政策摩擦
5. 同时做机械拆解和动态估计
6. 判断时间路径和持续性
7. 做行业和资产映射
8. 写出失效条件

### 3. 投资映射原则

分析完成后，必须把结论映射到以下层级中的至少两个层级：

1. 商品或现货方向
2. 行业与子行业
3. 股票与股票池
4. ETF 或指数工具
5. 利率、汇率或风险偏好资产
6. 需要回避的受损方向

映射时必须回答：

1. 谁直接受益
2. 谁直接受损
3. 谁可能被误伤
4. 哪些机会偏短期
5. 哪些机会偏中期
6. 结论的失效条件是什么

## 数据提供方设计

本 Skill 是公开分享版，不能假设用户拥有任何特定私有数据源。

### 数据提供方类别

可选数据提供方按职责分为：

1. 实时行情
2. 财务与估值
3. 选股与筛选
4. 行业与产业数据
5. 宏观统计
6. 政策与监管
7. 新闻与事件搜索
8. 网页抓取与原始来源读取

### 数据调用顺序

1. 用户已配置且可用的专属数据源
2. 插件内已安装且可用的公开适配器
3. 官方网站、监管披露、交易所、统计局等原始来源
4. 权威财经媒体和主流数据网站
5. 网页搜索兜底
6. 若仍无法确认，明确说无法验证，不硬推导

### 兜底原则

1. 兜底只能用于补充，不能替代关键数据验证
2. 涉及价格、政策、宏观、利率、汇率时，必须带日期与来源
3. 不同来源不一致时，优先原始来源，其次权威来源
4. 若时间戳不清或口径冲突，必须显式提示不确定性

## 数据验证硬规则

### 原则

凡是涉及以下数据，都必须先验证再推导：

1. 商品价格
2. 股票价格
3. 指数点位
4. 宏观数据
5. 政策发布日期
6. 利率与汇率
7. 财务数据与估值数据

### 必做检查

1. 数据是什么日期
2. 数据来自哪里
3. 数据单位是什么
4. 数据属于哪个市场
5. 这个数据是否就是用户问题中的那个口径

### 常见错误

1. 把历史数据当成当前数据
2. 混淆 A 股、港股、美股
3. 混淆现货、期货、主力合约、年度均价
4. 只引用二手新闻，不看原始公告或统计口径
5. 用价格结论替代产业逻辑

## 输出要求

### 标准输出结构

```markdown
# 主题

## 一、问题定义
- 事件或主题
- 分析口径
- 观察时间范围
- 核心要回答的问题

## 二、核心结论
- 先给三到五条高密度结论
- 明确方向、逻辑、持续性与风险

## 三、逻辑链条
- 冲击源
- 传导路径
- 关键变量
- 哪一层最先受影响
- 哪一层可能被钝化

## 四、数据与验证
- 关键数据
- 数据来源
- 时间戳
- 可信度
- 仍待验证的点

## 五、时间路径
- 短期
- 中期
- 更长周期
- 哪个阶段最重要

## 六、行业与资产映射
- 受益方向
- 受损方向
- 可能被误杀方向
- 观察指标

## 七、标的池
- A 股
- 港股
- ETF
- 商品或其他工具
- 每个标的对应一句逻辑

## 八、风险与失效条件
- 风险一
- 风险二
- 风险三

## 九、结论
- 一句话判断
- 现在最该跟踪的变量
```

### 输出风格要求

1. 先讲结论，再展开逻辑
2. 逻辑链必须完整，不跳步
3. 必须把分析和标的映射接起来
4. 不凭空给目标价
5. 不装作有实时数据
6. 对不确定部分明确说不确定

## 提问路由提示

以下表达通常应优先调用宏观供给冲击或事件驱动框架：

1. 这是不是一次再通胀交易
2. 油价上涨对中国通胀和 A 股影响是什么
3. 某个地缘冲突会不会推高 PPI 和 CPI
4. 某种资源涨价会不会扩散到全行业
5. 某个政策会不会改变供给格局
6. 帮我从事件推导到投资标的

## 质量要求

1. 逻辑链比结论更重要
2. 时间节奏必须明确
3. 至少给出两个反向验证点
4. 必须写失效条件
5. 必须区分事实、估算和判断

## 安全与边界

1. 本 Skill 用于研究与分析，不构成个性化投资建议
2. 不鼓励基于未经验证的数据做交易决策
3. 对高不确定性的结论，要缩小措辞强度
4. 若关键事实无法确认，应停止推导并说明原因
